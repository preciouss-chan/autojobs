                              DATA LOSS ANALYSIS
================================================================================

Resume with Typical Content:
  ✓ 1 person, 1-2 line summary
  ✓ 4 projects × 5-6 bullets each = 20-24 bullets total
  ✓ 3 experiences × 4-5 bullets each = 12-15 bullets total
  ✓ 20 total skills (7-8 languages, 6-7 frameworks, 6-7 tools)

After PDF Export (Single Page):
  ✗ Summary: 0 words (REMOVED)
  ✗ Projects: 6 bullets total (2-3 projects × 2-3 bullets each)
  ✗ Experience: 3-5 bullets total (likely 1-2 entries × bullets)
  ✓ Skills: 14 total (5 languages + 5 frameworks + 4 tools)

================================================================================
                            CRITICAL INSIGHTS
================================================================================

1. EXTRACTION IS COMPREHENSIVE
   ✓ Parse-resume extracts all resume content from PDF
   ✓ No limits in parsing pipeline
   ✓ Full data available in browser memory
   ✓ All data sent to API endpoints

2. EXPORT IS DESTRUCTIVE
   ✗ One-page PDF limit (757pt) forces truncation
   ✗ Summary always removed (unconditional)
   ✗ Bullets cascade truncated (3→2→1)
   ✗ Skills are ranked and best ones kept
   ✗ No user control over what's included

3. STORAGE DOESN'T EXIST
   ✗ No resume stored in database
   ✗ No resume history maintained
   ✗ Page refresh = must re-upload
   ✗ No version control for resumes

4. DISPLAY vs. EXPORT DIFFERENT
   ✓ UI preview shows: 5 exp bullets, 3 project bullets
   ✗ PDF export shows: 1-3 exp bullets, 3 project bullets
   ✓ Users see more in preview than what's exported

5. TAILORING DOESN'T MODIFY BULLETS
   ✓ Only summary is tailored
   ✓ Only skills are added/reframed
   ✓ Only cover letter is generated
   ✗ Bullets never modified (by design)
   ✗ Original bullets preserved intact
